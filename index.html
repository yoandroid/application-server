Servidor de Aplicaciones

Glosario

Red: una red o "network", es la tecnología que permite la comunicación entre computadoras, en otras palabras, que puedan enviar y recibir datos (LAN, WLAN, WAN, VLAN, Internet).

Paquetes: un paquete es un bloque de información que es enviado traves de las redes.

Protocolo: es el conjunto de reglas que las computadoras deben seguir y respetar para poder comunicarse (TCP/IP).

Latencia: es el tiempo de espera que tarda un paquete de datos en llegar a su destino.

Nodo: es una computadora conectada a una red, formando parte de ella.

Dirección IP: es un código numérico que identifica a una computadora dentro de una red.

Dominio: un dominio de internet es un nombre único fácil de encontrar que al ser procesado (Servidor DNS) se traduce en una dirección IP.

¿Qué es un servidor?

Físicamente, un servidor es una gran computadora que carece de perifericos, y en lugar de gabinetes, sus componentes son almacenados en "racks".

Conceptualmente, un servidor es un equipo diseñado para procesar solicitudes y "servir" información a otros equipos llamados "clientes".

Modelo Cliente-Servidor

El modelo cliente servidor es una arquitectura de software en el cual un dispositvo llamado cliente se comunica con el servidor a traves de peticiones. En este modelo, el servidor es quien maneja todos los recursos, y el que se encarga de estar a la "escucha" del cliente para procesar sus peticiones y entregarle una respuesta.

Servidor web

Servidor web: es un programa informático diseñado para responder las peticiones de un cliente. Los servidores web utilizan el protocolo HTTP para comunicarse con el cliente. La respuesta se entrega en forma de ficheros que pueden ser: documentos HTML, XML, PDF, imagenes JPG, PNG, SVG, videos MP4, AVI, WMV, audios MP3, WAV, WMA.

Keywords: petición, respuesta, HTTP, contenido estático.

Etapas del proceso de comunicación:

  • Cliente establece la conexión
  • El servidor resuelve la petición
  • Cliente interpreta la respuesta
  • Cierre de conexión

Servidor de aplicaciones

Servidor de aplicaciones: es un programa informático diseñado para ejecutar aplicaciones. Permite la interacción entre el usuario final/cliente y el código de la aplicación del lado del servidor, conocido comunmente como "lógica empresarial" (bussiness logic). La ejecución de ésta se traduce finalmente en contenido dinámico que se despliega en la página o aplicación web.

Keywords: lógica de negocio, contenido dinámico.

Comparación

Servidor web

  • Entrega contenido estático
  • Utiliza protocolos HTTP
  • Sirve solo aplicaciones basadas en la web
  • No tiene soporte para subprocesos múltiples, admite una solicitud a la vez
  • Facilita el tráfico web que no requiere muchos recursos

Servidor de aplicaciones

  • Entrega contenido dinámico
  • Utiliza diversos protocolos incluido HTTP, tales como CGI (conexión servidor web - aplicación externa)
  • Puede servir aplicaciones web y empresariales
  • Permite subprocesos múltiples, admite varias solicitudes en paralelo
  • Facilita procesos de ejecución más larga que consumen muchos recursos

Apache Tomcat

Apache Tomcat Official Page

¿Qué es?

  • Es un contenedor web con soporte de servlets y JSP
  • Desarrollado bajo el proyecto Jakarta
  • Desarrollado y actualizado por miembros de apache software foundation y voluntarios intdependientes

Ventajas

  • Es gratis y de código abierto
  • Instalación y configuración simple
  • Altamente escalable y adaptable a través de módulos
  • Las funciones integradas se utilizan para la autenticación y verificación del usuario
  • Se admiten lenguajes como Perl, PHP y Python, además de Java
